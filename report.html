Itemized Breakdown

  1. This site was created for Lukas Curtis, a client of my choosing.
  2. My site contains the header "Foxmosis"
  3. My site is technically one page with five scenes. A homepage, an About Me section, a Gallery, a Blog section, and a section which shows a specific blog rather than an overview
  4. My site contains navigation links to navigate between the different scenes
  5. My site has many different breakpoints to make sure content flows appropriately at all screen sizes, including phones
  6. My site contains a coherent style across all pages. Every page besides the homepage has the same navbar and some variant of the "colored brick" background. Each scene has it's own spreadsheet, and global rules (such as the navbar and the scene transitioning logic) have their own css spreadsheets as well
  7. All images on my site have an alt attribute to facilitate use of screen readers and webcrawlers. The image that is displayed in the gallery (when you click on an image) has it's alt property dynamically set as well to match the image that was selected
  8. My page contains a favicon
  9. Non-HTML elements are organized into folders. There is a folder for spreadsheets, another for scripts, and another one for stored images.
  10. No libraries were used to create the site. All content was produced by myself, Lukas Curtis, and background images were pulled from unsplash.com, a site that offers freely usable images to "power creators everywhere"
  11. My stylesheets pass all validation according to W3C's HTML validator.
  12. My file names are lowercase and alphanumeric
  13. My site is served out of port 8443
  14. My site is only accessible through HTTPS
  15. Content is neatly aligned, using flexbox as the primary tool to maintain alignment.
  16. My site contains a form to leave replies on blog posts.
  17. My site includes links to youtube, twitter, and instagram in the About Me section
  18. My source code uses conventional and consistent formatting and meaningful names for classes and IDs. CSS selectors that have a common context/container are given the same prefix. Selector id's and classes hint at the purpose/placement of the content
  19. My site uses an animation to switch between scenes, with a shaking garage-door effect
